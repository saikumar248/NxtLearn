Web Application

A web application is a collection of web resource programs having the capability to generate web pages.

We have two types of web pages:

Static Web Pages

A web page with fixed content is called a static web page.

A static web page is also known as a passive web page.

Examples:

●       Home page

●       About Us page

●       Contact page

●       Facebook login page

●       etc.

Dynamic Web Pages

A web page with dynamic content is called a dynamic web page.

A dynamic web page is also known as an active web page.

Examples:

●       Live cricket score page

●       Stock Market share value page

●       News page

●       etc.

We have two types of web resource programs:

Static Web Resource Programs

Static web resource programs are used to generate static web pages.

Examples:

●       HTML program

●       CSS program

●       Bootstrap program

●       AngularJS program

●       etc.

Dynamic Web Resource Programs

Dynamic web resource programs are used to generate dynamic web pages.

Examples:

●       Servlet program

●       JSP program

●       etc.

Based on the position and execution, these web resource programs are divided into two types:

Client-side Web Resource Programs

A web resource program that executes at the client-side is called a client-side web resource program.

All static web resource programs are called client-side web resource programs.

Examples:

●       HTML program

●       CSS program

●       Bootstrap program

●       AngularJS program

●       etc.

Server-side Web Resource Programs

A web resource program that executes at the server-side is called a server-side web resource program.

All dynamic web resource programs are called server-side web resource programs.

Examples:

●       Servlet program

●       JSP program

●       etc.

Web application and Web Resource program execution

Java program will execute manually.

Web application and web resource program will execute when we give the request.

With respect to the diagram

  1. Enduser gives the request to web resource program.
  2. Webserver traps that request and passes that request to appropriate web resource program.
  3. Web resource program will execute the logic to process the request.
  4. Web resource program communicates with database software if necessary.
  5. Web resource program gives output to web server.
  6. Web server send the output to browser window as dynamic response.

Note:

The process of keeping the application in a server is called deployment and reverse is called undeployment.

Web Server

It is an installable software which is used to automate the whole process of web application and web resource program execution. Examples:

●       Tomcat

●       Resin

Responsibilities of web server

  1. It takes continuous requests from the client.
  2. It passes the request to the appropriate web resource program.
  3. It provides the environment to deploy and undeploy the web application.
  4. It allows client-side web resource programs to execute in a browser window.
  5. It will add middleware services only to deployed web applications.
  6. It gathers the result and sends it to the browser window as a dynamic response.
  7. It is used to automate the whole process of web application and web resource program execution.

To execute a Java program, we require a JRE/JVM. To execute a servlet program, we require a servlet container. To execute a JSP program, we require a JSP container.

Web Container

A web container is a software application or a program which is used to manage the whole life cycle of a web resource program, i.e., from birth to death.

Servlet container manages the whole life cycle of a servlet program. JSP container manages the whole life cycle of a JSP program.

Some parts of the industry consider servlet containers and JSP containers as web containers.

Every server is designed to support servlet containers and JSP containers, so we don't need to arrange them separately.

package com.ihub.www;

import java.io.IOException;
import java.io.PrintWriter;
import java.util.Date;

import javax.servlet.GenericServlet;
import javax.servlet.ServletException;
import javax.servlet.ServletRequest;
import javax.servlet.ServletResponse;

public class DateSrv extends GenericServlet
{
public void service(ServletRequest req, ServletResponse res) throws ServletException, IOException
{
PrintWriter pw = res.getWriter();
res.setContentType("text/html");

Date d = new Date();

pw.println("<center><h1>Current Date and Time <br> " + d + " </h1></center>");

pw.close();
}
}

URL Patterns in Servlets

URL patterns hide servlet name and technology details from outsiders for security reasons. Clients, web resources, and web servers recognize servlet programs using URL patterns.

Types of URL Patterns

1. Exact Match URL Pattern

●       Starts with '/' followed by a name

●       Example in web.xml:

<url-pattern>/test</url-pattern>

Request URL examples:

●       Valid: http://localhost:2525/DateApp/test

●       Invalid: http://localhost:2525/DateApp/best

●       Invalid: http://localhost:2525/DateApp/x/test

2. Directory Match URL Pattern

●       Starts with '/' and ends with '*'

●       Example in web.xml:

<url-pattern>/x/y/*</url-pattern>

Request URL examples:

●       Valid: http://localhost:2525/DateApp/x/y/z

●       Valid: http://localhost:2525/DateApp/x/y/z/test

●       Invalid: http://localhost:2525/DateApp/y/x/z

3. Extension Match URL Pattern

●       Starts with '*' having some extension

●       Example in web.xml:

<url-pattern>*.do</url-pattern>

Request URL examples:

●       Invalid: http://localhost:2525/DateApp/temp

●       Valid: http://localhost:2525/DateApp/temp.do

●       Valid: http://localhost:2525/DateApp/x/y/z.do

MIME Types

MIME (Multipurpose Internet Mail Extension) describes various output formats in servlets:

  1. text/html: Display output in HTML format
  2. text/xml: Display output in XML format
  3. application/ms-word: Display output in Word format
  4. application/vnd.ms-excel: Display output in Excel format

package com.ihub.www;

import java.io.*;
import java.util.Calendar;
import javax.servlet.*;
import javax.servlet.http.*;

public class WishSrv extends HttpServlet {
public void service(HttpServletRequest req, HttpServletResponse res)
throws ServletException, IOException {
PrintWriter pw = res.getWriter();
res.setContentType("text/html");

Calendar c = Calendar.getInstance();
int h = c.get(Calendar.HOUR_OF_DAY);

if(h < 12)
pw.println("<center><h1>Good Morning</h1></center>");
else if(h < 16)
pw.println("<center><h1>Good Afternoon</h1></center>");
else if(h < 20)
pw.println("<center><h1>Good Evening</h1></center>");
else
pw.println("<center><h1>Good Night</h1></center>");

pw.close();
}
}

package com.ihub.www;

import java.io.IOException;
import java.io.PrintWriter;
import javax.servlet.ServletException;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

public class VoteSrv extends HttpServlet {
protected void doGet(HttpServletRequest req, HttpServletResponse res)
throws ServletException, IOException {
PrintWriter pw = res.getWriter();
res.setContentType("text/html");

// Reading form data
String name = req.getParameter("t1");
String sage = req.getParameter("t2");

// Converting string age to int
int age = Integer.parseInt(sage);

if(age < 18)
pw.println("<center><h1 style='color:red;'>" + name + " You are not eligible to vote</h1></center>");
else
pw.println("<center><h1 style='color:green;'>" + name + " You are eligible to vote</h1></center>");

pw.close();
}
}

package com.ihub.www;

import java.io.*;
import java.sql.*;
import javax.servlet.*;
import javax.servlet.http.*;
public class DBSrv extends HttpServlet {
protected void doGet(HttpServletRequest req, HttpServletResponse res)
throws ServletException, IOException {
PrintWriter pw = res.getWriter();
res.setContentType("text/html");

// Reading form data
String sno = req.getParameter("t1");
int no = Integer.parseInt(sno);
String name = req.getParameter("t2");
String add = req.getParameter("t3");

// Database operations
try {
Class.forName("oracle.jdbc.driver.OracleDriver");
System.out.println("Oracle JDBC Driver Loaded Successfully!");
} catch (ClassNotFoundException e) {
System.out.println("Failed to load Oracle JDBC Driver.");
e.printStackTrace();
}
try (Connection con = DriverManager.getConnection(
"jdbc:oracle:thin:@localhost:1521:XE", "system", "admin")) {

// Class.forName("oracle.jdbc.driver.OracleDriver");
String qry = "insert into student values(?,?,?)";

try (PreparedStatement ps = con.prepareStatement(qry)) {
ps.setInt(1, no);
ps.setString(2, name);
ps.setString(3, add);

int result = ps.executeUpdate();
String message = (result == 0) ? "No Record Inserted" : "Record Inserted";
pw.println("<center><h1>" + message + "</h1></center>");
}
} catch(Exception e) {
e.printStackTrace();
} finally {
pw.close();
}
}
}

Servlet Life Cycle Methods

There are three core lifecycle methods in servlets:

1) init(ServletConfig config)

public void init(ServletConfig config) throws ServletException

●       Used for servlet instantiation

●       Executes just before servlet object creation

2) service(ServletRequest req, ServletResponse res)

public void service(ServletRequest req, ServletResponse res) throws ServletException, IOException

●       Handles request arrival events

●       Executes whenever a request is made to the servlet

3) destroy()

public void destroy()

●       Handles destruction events

●       Executes just before servlet object destruction

Servlet Filters

Filter is an object which is executed at the time of preprocessing and postprocessing of the request.

●       To count number of request coming to the application

●       To perform form validation

●       To perform encryption and decryption

●       Other preprocessing/postprocessing tasks

Like Servlet, Filter also having it own API called Filter API.

Filter API Interfaces

The javax.servlet package provides three main interfaces:

  1. Filter
  2. FilterChain
  3. FilterConfig

1) Filter Interface Methods

For creating any filter, we must and should implements the Filter interface.

Filter interface provides the following 3 life cycle methods for filter.

 

i) init(FilterConfig config)

public void init(FilterConfig config) throws ServletException

●       Initializes the filter

●       Invoked once at start

ii) doFilter(HttpServletRequest req, HttpServletResponse res, FilterChain chain)

public void doFilter(HttpServletRequest req, HttpServletResponse res, FilterChain chain)

●       Invoked for each request to filtered resources

●       Performs the actual filtering

iii) destroy()

public void destroy()

●       Invoked once when filter is removed from service

2) FilterChain

It is responsible to invoke the next filter or resource in the chain.

FilterChain contains only one method.

i) doFilter(HttpServletRequest req,HttpServletResponse res)

public void doFilter(HttpServletRequest req,HttpServletResponse res)

It passes the control to the next filter or resource.

 

3)FilterConfig

For every filter our servlet container creates FilterConfig object.

It is one per filter.

Servlet to Servlet Communication (Servlet Chaining)

There are three main ways to achieve servlet to servlet communication:

  1. Request Forwarding
  2. Response Including
  3. Send Redirection

1. Request Forwarding

In request forwarding, the output of the source program is discarded and only the output of the destination program appears in the browser window as a dynamic response.

To forward a request:

  1. Use RequestDispatcher object (from javax.servlet package)
  2. Create it using getRequestDispatcher() method of HttpServletRequest

RequestDispatcher rd = req.getRequestDispatcher();
rd.forward(req, res);

2. Including the Response

When including the response, both the source servlet and destination servlet outputs are combined and sent to the browser window as a dynamic response.

To include a response:

  1. Use RequestDispatcher object (from javax.servlet package)
  2. Create it using getRequestDispatcher() method of HttpServletRequest

RequestDispatcher rd = req.getRequestDispatcher();
rd.include(req, res);
